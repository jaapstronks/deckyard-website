# Comments

Provide feedback and collaborate through slide comments.

## Overview

Comments let team members discuss specific slides, suggest changes, and track decisions. Use comments to gather feedback without modifying the presentation content.

![Slide comment thread with Reply, Resolve, and Delete actions alongside the live preview and presenter notes](/images/screenshots/comments-with-thread.png)

## Adding Comments

### On a Slide

1. Navigate to the slide you want to comment on
2. Click the **Comments** icon in the toolbar (or press `C`)
3. Type your comment in the text field
4. Click **Post** or press `Enter`

### On Specific Content

To comment on a specific element:
1. Select the element (text, image, etc.)
2. Right-click and select **Add Comment**
3. Type your comment
4. Click **Post**

The comment is anchored to that element.

### Quick Comment

Use keyboard shortcuts for faster commenting:
1. Press `C` to open comment mode
2. Type your comment
3. Press `Enter` to post

## Comment Panel

The comment panel shows all comments for the current presentation.

![Comments panel with filters, thread options, and sample comment](/images/screenshots/comments-panel-new.png)

### Opening the Panel

- Click the **Comments** icon in the toolbar
- Use the keyboard shortcut `C`
- Click on a comment indicator on a slide

### Panel Features

- **Filter by slide** - Show comments for current slide only
- **Filter by status** - Show open, resolved, or all comments
- **Sort order** - Newest first or oldest first
- **Search** - Find comments by text

### Comment Threads

Each comment can start a discussion thread:
1. Click on a comment to expand it
2. Type a reply in the reply field
3. Click **Reply**

Threads keep related discussions organized.

## Resolving Comments

When feedback has been addressed:

1. Click on the comment
2. Click **Resolve**
3. Comment is marked as resolved and faded

### Resolved Comments

Resolved comments:
- Remain visible but are de-emphasized
- Can be filtered out of the view
- Can be reopened if needed

### Reopening Comments

To reopen a resolved comment:
1. Find the resolved comment
2. Click **Reopen**
3. Comment becomes active again

### Bulk Actions

Resolve multiple comments at once:
1. Select multiple comments (Shift+click)
2. Click **Resolve Selected**

## Mentions and Notifications

When you mention a team member in a comment, they'll receive a notification.

### Mentioning Collaborators

To notify someone:
1. Type `@` in your comment
2. Start typing their name
3. Select from the autocomplete list
4. They receive a notification

### Notification Behavior

When mentioned, users receive:
- **In-app notification** - Badge on the bell icon
- **Email notification** - If enabled in their settings
- **Direct link** - Click to go straight to the comment

### Notification Settings

Users can configure their notification preferences:
1. Go to **Settings** > **Notifications**
2. Toggle email notifications on/off
3. Set notification frequency (immediate, daily digest)

## Comment Features

### Formatting

Comments support basic formatting:
- **Bold** - Wrap text in `**asterisks**`
- *Italic* - Wrap text in `*single asterisks*`
- `Code` - Wrap text in backticks
- Links - Paste URLs, they auto-link

### Suggestions

AI-generated suggestions appear as special comments:
- Marked with a suggestion indicator
- Include the proposed change
- One-click to apply or dismiss

### Action Items

Mark comments as action items:
1. Create a comment
2. Check the **Action Item** box
3. Optionally assign to someone

Action items can be tracked and filtered separately.

## Comment Visibility

### Who Can See Comments

- All collaborators with comment or edit permission
- Comments are private to the collaboration team
- Public viewers cannot see comments

### Comments Are Not Exported

Comments are not included in:
- Published presentations
- PDF exports
- PowerPoint exports
- Public share links

## Keyboard Shortcuts

| Shortcut | Action |
|----------|--------|
| `C` | Open/close comment panel |
| `Enter` | Post comment |
| `Shift + Enter` | New line in comment |
| `Esc` | Cancel comment |

## Best Practices

### Be Specific

Good: "The Q3 revenue figure should be $2.3M, not $2.1M"
Bad: "This number is wrong"

### Use Threads

- Reply to existing comments instead of creating new ones
- Keeps related discussion together
- Makes it easier to follow conversations

### Resolve Promptly

- Mark comments resolved when addressed
- Add a brief note about what was done
- Keeps the comment list manageable

### Use Mentions Wisely

- Mention people when you need their input
- Don't over-mention (notification fatigue)
- Use for questions and decisions
